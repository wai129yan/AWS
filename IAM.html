Use and enforce the use of MFA - Create and use Roles for giving permissions to AWS services - Use Access Keys for Programmatic Access (CLI / SDK) - Never share IAM users & Access Keys • Users: mapped to a physical user, has a password for AWS Console • Groups: contains users only • Policies: JSON document that outlines permissions for users or groups • Roles: for EC2 instances or AWS services • Security: MFA + Password Policy • AWS CLI: manage your AWS services using the command-line • AWS SDK: manage your AWS services using a programming language • Access Keys: access AWS using the CLI or SDK • Audit: IAM Credential Reports & IAM Access Advisor Memory Optimize in AWS (မြန်မာဘာသာဖြင့်) AWS တွင် မှတ်ဉာဏ်အသုံးချမှု အကောင်းဆုံးဖြစ်စေရန် အောက်ပါအချက်များကို လုပ်ဆောင်နိုင်ပါသည်: • မှတ်ဉာဏ်များသော EC2 instance အမျိုးအစားများကို အသုံးပြုခြင်း (ဥပမာ - R5, X1, High Memory instances) • ElastiCache ကိုအသုံးပြု၍ in-memory caching ပြုလုပ်ခြင်း • Amazon RDS for Aurora ကဲ့သို့သော မှတ်ဉာဏ်အသုံးချမှု အကောင်းဆုံးဖြစ်အောင် ပြုလုပ်ထားသော database များကို အသုံးပြုခြင်း • Lambda function များအတွက် မှတ်ဉာဏ်ပမာဏကို သင့်လျော်စွာ သတ်မှတ်ပေးခြင်း • Amazon EMR cluster များတွင် မှတ်ဉာဏ်အသုံးချမှုကို ကောင်းမွန်စွာ စီမံခန့်ခွဲခြင်း • CloudWatch မှတဆင့် မှတ်ဉာဏ်အသုံးပြုမှုကို စောင့်ကြည့်စစ်ဆေးခြင်းနှင့် လိုအပ်သလို ချိန်ညှိခြင်း ဤနည်းလမ်းများဖြင့် AWS ပေါ်တွင် မှတ်ဉာဏ်အသုံးချမှုကို ပိုမိုကောင်းမွန်စေနိုင်ပါသည်။
